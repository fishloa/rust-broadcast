# `vvcC` — VvcDecoderConfigurationRecord (ISO/IEC 14496-15:2022 §11.3.3)

The ISO text is paid and not vendored. The `vvcC` **byte layout is a data-format
fact** defined by 14496-15:2022 §11 — described below in our own ISO-style
syntax notation. It was **cross-validated** (byte-for-byte) against the `vvcC`
box that FFmpeg's muxer emits into our fixture `fixtures/mp4/frag/vvc.frag.mp4`,
and against the field set of the open reference implementations
(FFmpeg `libavformat/vvc.c`, GPAC `GF_VVCConfig`). No third-party source code is
reproduced here — this is an independent description of the wire format,
verified against reference *output*.

Carriage: `vvc1` (parameter sets in `vvcC`) / `vvi1` (in-band) ISOBMFF sample
entry; TS `stream_type` 0x33. NAL length field width = `lengthSizeMinusOne + 1`.

## Byte layout (big-endian; bit fields MSB-first)

```
aligned(8) class VvcDecoderConfigurationRecord {
  // byte 0:
  bit(5)  reserved = '11111'
  bit(2)  LengthSizeMinusOne
  bit(1)  ptl_present_flag

  if (ptl_present_flag) {
    bit(9)  ols_idx
    bit(3)  num_sublayers
    bit(2)  constant_frame_rate
    bit(2)  chroma_format_idc

    bit(3)  bit_depth_minus8
    bit(5)  reserved = '11111'

    // ---- VvcPTLRecord ----
    bit(2)  reserved = '00'
    bit(6)  num_bytes_constraint_info
    bit(7)  general_profile_idc
    bit(1)  general_tier_flag
    bit(8)  general_level_idc
    // the following flags+bytes are bit-packed, then padded to a byte boundary:
    bit(1)  ptl_frame_only_constraint_flag
    bit(1)  ptl_multilayer_enabled_flag
    bit(8 * (num_bytes_constraint_info-1))  general_constraint_info bytes
    bit(6)  final 6 bits of general_constraint_info[last]
    // flush to byte boundary

    if (num_sublayers > 1) {
      // one byte: ptl_sublayer_level_present_flag[num_sublayers-2 .. 0] packed
      //   (ffmpeg builds it MSB=high sublayer; remaining bits 0)
      bit(8)  ptl_sublayer_level_present_flags
    }
    for (i = num_sublayers-2 .. 0)
      if (ptl_sublayer_level_present_flag[i]) bit(8) sublayer_level_idc[i]

    bit(8)  ptl_num_sub_profiles
    for (j = 0 .. ptl_num_sub_profiles-1)
      bit(32) general_sub_profile_idc[j]
    // ---- end VvcPTLRecord ----

    bit(16) max_picture_width
    bit(16) max_picture_height
    bit(16) avg_frame_rate
  }

  bit(8)  num_of_arrays
  for (i = 0 .. num_of_arrays-1) {
    bit(1)  array_completeness
    bit(2)  reserved = '00'
    bit(5)  NAL_unit_type          // 14=DCI,15=VPS,16=SPS,17=PPS,18=prefix APS,…
    bit(16) num_nalus              // (== 1 implied for DCI/OPI arrays)
    for (j = 0 .. num_nalus-1) {
      bit(16) nal_unit_length
      bit(8*nal_unit_length) nalUnit
    }
  }
}
```

## Notes
- `ptl_present_flag` is essentially always 1 for `vvc1`.
- `constant_frame_rate`: 0=unknown/variable, 1=constant, 2=constant-per-temporal-subset.
- `chroma_format_idc` / `bit_depth_minus8` mirror the SPS values (all SPS in the
  record must agree, per §11).
- `max_picture_width`/`height` = the SPS `sps_pic_width/height_max_in_luma_samples`.
- Arrays carry the VPS/SPS/PPS (and optionally DCI/OPI/APS) NAL units, exactly the
  `hvcC` array shape (`avcC-hvcC-14496-15.md`).
- rfc6381: `vvc1.<general_profile_idc>.<S|L><general_level_idc>.<constraint hex>` per
  the VVC codecs-parameter registration.

Round-trip requirement (project invariant): parse → serialize must reproduce the
source `vvcC` bytes exactly; the value is byte-verified against ffmpeg's box in
`vvc.frag.mp4`.
