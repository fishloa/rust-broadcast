# ADR-0001: Macro-driven, drift-tested dispatch for all tag/type enums

- **Status:** Accepted (2026-06-10)
- **Scope:** all `dvb-*` crates

## Context

The workspace turns a wire discriminant into a typed value at several points:

| Discriminant | Dispatch enum | Crate |
|---|---|---|
| `descriptor_tag` | `AnyDescriptor` | dvb-si |
| `descriptor_tag_extension` (within tag `0x7F`) | extension bodies | dvb-si |
| `table_id` | `AnyTableSection` | dvb-si |
| `packet_type` | `AnyPayload` | dvb-t2mi |

Three of these (`AnyDescriptor`, `AnyTableSection`, `AnyPayload`) are generated
by a `declare_*!` macro from a single list. One line per type produces the enum
variant, the `From<T>` impl, the dispatcher arm, and a **drift test** that pins
each tag/`table_id`/`packet_type` literal to the type's `*Def` trait const
(`DescriptorDef::TAG`, etc.). The list is the single source of truth, so the
dispatcher can never silently drift from the implemented set â€” adding a type is
one line, and a mismatched literal fails the build/tests.

The **extension-descriptor sub-bodies** (`descriptor_tag_extension` â†’
`ExtensionBody`) were the exception: hand-rolled as several parallel matches
that must be edited together by hand â€” the `ExtensionTag` enum, a `kind()`
match, a `parse_body()` match, the `ExtensionBody` enum, and the `Serialize`
arms â€” with **no drift test**. Adding a body touched ~4 lists; nothing enforced
their mutual consistency (the `kind()`/`parse_body()` arm counts had already
diverged, harmlessly, via named-but-`Raw` bodies). As the typed body set grew
(#53), this hand-maintained surface and its drift risk grew with it.

## Decision

**Every wire-discriminant dispatch enum in the workspace uses the macro-driven,
drift-tested pattern. No hand-rolled parallel dispatch lists.**

Concretely for the extension level:

1. Each extension body is a first-class type implementing `Parse<'a>` +
   `Serialize` + an `ExtensionBodyDef` trait (same shape as `DescriptorDef`:
   `TAG` = the `descriptor_tag_extension` value, `NAME` = SCREAMING_SNAKE).
2. The dispatch is generated by the **same** macro as `AnyDescriptor`,
   generalised to be parameterised over the enum name, the `*Def` trait, and the
   fall-through variant(s) â€” top-level descriptors fall through to
   `Other`/`Unknown`, extension bodies to `Raw(&[u8])`. A new body is one macro
   line; the drift test is generated automatically.

New bodies and new descriptors/tables/payloads are added the same way
everywhere: one line in the macro list, type implements the `*Def` trait.

## Consequences

- **+** One dispatch mechanism across all `dvb-*` crates; consistent to read and
  extend.
- **+** Drift is a compile/test-time error everywhere, including the extension
  level that previously had none.
- **+** Adding a body/descriptor/table/payload is a single line.
- **âˆ’** Each extension body must become a uniform `Parse`/`Serialize` + `*Def`
  type (some were inline / free `parse_*` fns), and the existing hand-rolled
  extension dispatch is a one-time refactor (~17 bodies). Tracked alongside the
  #53 sweep; bodies added before the refactor are migrated into the macro.
- **Future (optional):** hoist the generalised dispatch macro into `dvb-common`
  so `AnyDescriptor`, `AnyExtensionBody`, `AnyTableSection`, and `AnyPayload`
  share one definition instead of three near-identical per-crate copies. A
  follow-up ADR would record that if taken.
