# ts-fix v0.1.0 — MPEG-2 TS repair / remux (container layer)

> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development. Each task = one operation, gated by a fault-inject→repair→assert-known-good test on a REAL capture. Build on mpeg-ts + dvb-si. No codec parsing.

**Goal:** A `ts-fix` crate (GitHub #357) that repairs/remuxes MPEG-2 TS at the container/timing layer — never touching codec bitstreams.

**Architecture:** A forward-compatible **builder-driven streaming engine**. Feed 188-byte packets → emit repaired packets; `finish()` flushes. The engine owns operation ordering internally; each operation is opt-in via a builder method. Adding future ops = additive builder methods (no breaking change). All config/error enums `#[non_exhaustive]`.

**Tech stack:** mpeg-ts (framing, adaptation/PCR, iter_packets, mux packetizer), dvb-si (PAT/PMT parse for PID-follow + regen). MSRV 1.81. Cites only ISO/IEC 13818-1 (owned/free).

## Global constraints (verbatim)

- **Forward-compat is the hard requirement (owner: "no leroy").** The v0.1 public API must accommodate all 8 eventual ops so v0.2/v0.3 additions are ADDITIVE, never breaking. NO public `enum Operation` (breaks on new variant), NO public `trait Operation` (locks the contract before PES-reconstruction's needs are known). Builder methods + `#[non_exhaustive]` configs only.
- Each op gated by a **fault-inject → repair → assert-known-good** test on a REAL capture (oracle = the clean original or a TSDuck reference). A plain round-trip is NOT sufficient.
- No magic numbers outside `#[cfg(test)]`; module docs cite ISO/IEC 13818-1 §.
- `--no-default-features` builds; no_std-friendly where practical (CLI is std).
- CLI follows docs/CLI-STANDARD.md (clap).
- Symmetric serialize + round-trip invariant where types are introduced.

## v0.1.0 scope: 5 operations

CC repair, PID filter / service extract, PAT/PMT regen, null stuffing/drop, **PCR restamp**. (Deferred to later: PTS/DTS wrap, PES AU reconstruction, SCTE-35 preservation.)

## Fixture strategy (baked into every op gate)

1. **Fault injection on real captures** — a committed deterministic helper corrupts a real clean capture (mpeg-ts `tests/fixtures/m6-single.ts` for single-program; fetch-on-demand `france-tnt-uhf32.ts`/`hotbird-mhp.ts` for multi-program) in the exact way the op fixes; oracle = the clean original / a known invariant.
2. **TSDuck input→reference pairs** (`.test-streams/tsduck-test/{input,reference}/`) where a `tsp` plugin overlaps the op (continuity, filter, pcr) — third-party ground truth.
3. Small corrupted slices committed for hermetic CI; large multi-program via fetch-on-demand (#67).

```rust
// builder — every op is opt-in; engine orders internally; new ops = new methods (additive)
pub struct TsFix { /* private */ }
pub struct TsFixBuilder { /* private */ }
impl TsFix {
    pub fn builder() -> TsFixBuilder;
    /// Feed one 188-byte packet; emitted (repaired) packets are handed to `out`.
    pub fn push(&mut self, packet: &[u8], out: impl FnMut(&[u8]));
    /// Flush buffered state at end of stream.
    pub fn finish(&mut self, out: impl FnMut(&[u8]));
}
impl TsFixBuilder {
    pub fn repair_continuity(self) -> Self;          // Task 2

    pub fn restamp_pcr(self, cfg: PcrRestamp) -> Self; // Task 6
    pub fn build(self) -> Result<TsFix>;
}
#[non_exhaustive] pub enum Error { /* BufferTooShort, … */ }
```

Engine applies ops in a fixed correct order (filter → regen_psi → cc_repair → pcr_restamp → stuffing). Internal buffering allowed (so future PES-reconstruction ops fit without API change).

## Task 2: CC repair

**Files:** `ts-fix/src/ops/continuity.rs`; test `ts-fix/tests/cc_repair.rs`; fixture helper `ts-fix/tests/support/corrupt.rs`.

Renumber `continuity_counter` per PID to a correct monotonic sequence (mod 16), respecting the "no increment on adaptation-only / duplicate" rules (ISO 13818-1 §2.4.3.3). 

- [ ] Fault-inject: read m6-single.ts, scramble/zero the CC field per packet. 
- [ ] Repair → assert per-PID CC is correct monotonic (increment only on payload-bearing packets, mod 16) AND for the reversible case byte-matches the original capture's CC sequence.
- [ ] Boundary: a PID with ≥2 packets interleaved with another PID (forces per-PID state, not global counter).
- [ ] Gate: `cargo test -p ts-fix --test cc_repair`.

## Task 3: PID filter / service extract

**Files:** `ts-fix/src/ops/pid_filter.rs`; `ts-fix/tests/pid_filter.rs`.

`PidFilter { keep: PidSet }` or `service: u16` (extract one program: keep its PMT PID + all ES PIDs from the PMT + PAT + PCR PID). Uses dvb-si to read PAT/PMT to resolve a service's PIDs.

**Interfaces produced:** `#[non_exhaustive] pub struct PidFilter { … }` + constructors `PidFilter::keep(pids)` / `PidFilter::service(program_number)`.

- [ ] Fixture: real multi-program mux (france-tnt-uhf32.ts via fetch-on-demand).
- [ ] Extract one service → assert output contains ONLY that service's PIDs (+ PAT/PMT/PCR), all others dropped; PMT still parses; PAT lists only the kept program (coordinate with Task 4).
- [ ] Gate: `cargo test -p ts-fix --test pid_filter`.

## Task 4: PAT/PMT regen

**Files:** `ts-fix/src/ops/psi_regen.rs`; `ts-fix/tests/psi_regen.rs`.

Rebuild PAT (and optionally PMT) so the program tables are consistent with the actual streams present (e.g. after PID filtering, PAT lists only surviving programs). Uses dvb-si Pat/Pmt builders + mpeg-ts SectionPacketizer.

- [ ] Fault-inject: strip/corrupt the PAT in m6-single.ts → regen → assert a valid PAT with correct CRC listing the present program(s); section parses via dvb-si.
- [ ] Combined with Task 3: after service-extract, regen PAT → PAT lists exactly the one kept program.
- [ ] Gate: `cargo test -p ts-fix --test psi_regen`.

## Task 6: PCR restamp (the forward-compat anchor)

**Files:** `ts-fix/src/ops/pcr_restamp.rs`; `ts-fix/tests/pcr_restamp.rs`.

Recompute the 42-bit PCR in adaptation fields (mpeg-ts already parses `PcrValue`) on the PCR PID, based on a timing model (packet position × bitrate, or interpolation between observed PCRs). The `StreamModel` timing context introduced here is what PTS/DTS-wrap (v0.2) will reuse — design it general (a `TimingContext` carrying clock rate + last-PCR state), NOT inline to PCR only.

**Interfaces produced:** `#[non_exhaustive] pub struct PcrRestamp { … }` (mode: interpolate | from_bitrate).

- [ ] Fault-inject: perturb PCR values in m6-single.ts (add offset/jitter) → restamp → assert PCR is monotonic non-decreasing on the PCR PID AND within tolerance of the original PCRs (oracle = original capture's PCR timeline).
- [ ] Assert the PCR field bytes round-trip (mpeg-ts PcrValue serialize) — no raw passthrough.
- [ ] Gate: `cargo test -p ts-fix --test pcr_restamp`.
