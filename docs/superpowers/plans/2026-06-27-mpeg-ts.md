# mpeg-ts Extraction Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract dvb-si's generic MPEG-2 TS framing layer into a new standalone `mpeg-ts` crate; dvb-si then depends on it and re-exports the moved types (non-breaking).

**Architecture:** `git mv` the 5 generic files (`ts`/`section`/`resync`/`mux`/`pid`) from `dvb-si/src` into `mpeg-ts/src` (history preserved); give `mpeg-ts` its own `error` module whose variants match what the moved code uses, so `crate::` rebinds cleanly with near-zero edits. `dvb-si` gains an `mpeg-ts` dependency, re-exports the moved modules under their existing paths, rebuilds the SI-aware `SiDemux` (`demux.rs`, stays) on `mpeg_ts::` primitives, and maps `mpeg_ts::Error` at the boundary. Adds an owned `TsPacketBuf` (pipeline ergonomics, seeded from zenith).

**Tech Stack:** Rust workspace, `dvb-common` (Parse/Serialize, CRC-32/MPEG-2), `thiserror`, optional `serde`. Spec: ITU-T H.222.0 (08/2023) = ISO/IEC 13818-1, vendored at `specs/itu_t_h222_0_202308_mpeg2_systems.pdf`; transport_packet/adaptation_field transcription at `mpeg-ts/docs/tables/h222_0/transport_packet.md`.

## Global Constraints

- MSRV **1.81**; build/test `--locked`. CI `RUSTFLAGS="-D warnings"`. 6 gates + MSRV + bare-metal `thumbv7em-none-eabi` no_std.
- `#![no_std]` + `alloc`; `serde` feature-gated. `#![forbid(unsafe_code)]` (matches dvb-si TS layer).
- **No breaking change to dvb-si's public API** — `dvb_si::ts::*` / `::section::*` / `::resync::*` / `::mux::*` / `::pid::*` must still resolve (re-exports of `mpeg_ts::*`).
- Round-trip invariant travels with the code: every parse has a symmetric serialize + byte-identical round-trip test.
- No magic numbers outside `#[cfg(test)]`. Every public enum: `name()` + `impl_spec_display!` + `label_coverage` guard. `#[non_exhaustive]` on public enums.
- Module `//!` cites **ITU-T H.222.0 §2.4 (= ISO/IEC 13818-1)** for the framing layouts.
- `mpeg-ts` = new crate, **v0.1.0**, independent. dvb-si's new dep forces a **lockstep v7.10.0** (all 6 core crates) — bundle the already-on-main `dvb-tools` clap-CLI work (#344).

> Seed from zenith's `zenith-ts/src/packet.rs` `OwnedTsPacket` (read it for reference — the `payload()`/`payload_mut()` offset logic and `serialize_with_payload` are proven there). Build the header parse on `crate::ts::TsHeader::parse` (don't duplicate bit-twiddling). Do NOT bring zenith's ISI/parse-stat/BBFrame fields — those are pipeline-specific; keep `TsPacketBuf` generic.

- [ ] **Step 1: Write the failing test** (in `packet_buf.rs` `#[cfg(test)]`)

```rust
#[cfg(test)]
mod tests {
    use super::*;
    #[test]
    fn owned_round_trip_and_payload_mut() {
        let payload = [0xAAu8; 184];
        let mut pkt = TsPacketBuf::parse(TsPacketBuf::serialize_with_payload(0x0100, true, 7, &payload)).unwrap();
        assert_eq!(pkt.pid, 0x0100);
        assert!(pkt.pusi);
        assert_eq!(pkt.continuity_counter, 7);
        assert_eq!(pkt.payload().unwrap()[..184], payload[..]);
```
