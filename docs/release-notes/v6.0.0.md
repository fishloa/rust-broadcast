# 6.0.0 — decode completeness

5.0 *typed* every wire field; 6.0 *decodes* them. The defining promise of these
crates is that a consumer gets spec-faithful, decoded data and never
re-implements an ETSI lookup table — yet 5.0 still handed back bare integers for
spec-enumerated codes (`running_status`, `service_type`, `stream_type`, content
genre, …). Every downstream then re-built Table 6 / Table 87 / Table 28 by hand.
6.0 closes that gap.

## What changed

### Coded fields are typed enums (breaking)

A `u8` the spec assigns named meanings is, in API terms, the same sin as a raw
`&[u8]` — one level up. So the single coded fields now carry their enum:
`SdtService.running_status` is a `RunningStatus`, `PmtStream.stream_type` is a
`StreamType`, `ServiceDescriptor.service_type` is a `ServiceType`, and so on
across the descriptor, table, and delivery-system surface.

Each enum is **lossless**: a `Reserved(u8)` / `UserDefined(u8)` catch-all carries
any unallocated wire value verbatim, so `parse → serialize` stays byte-identical
(the hard project invariant) and no information is dropped. Each exposes
`from_u8`/`to_u8` + `name()`.

### Decode accessors where a field can't become an enum (additive)

Content genre spans two nibbles, component_type a three-field triple, and the CA /
private-data-specifier / data-broadcast IDs are large living registries — none of
those collapse to one enum, so they get methods instead: `ContentEntry::genre()`
/ `genre_name()`, `epg::Rating::minimum_age()`, AC-3/E-AC-3/AAC component decode,
and best-effort `ca_system_name()` / `private_data_specifier_name()` /
`data_broadcast_id_name()` lookups (documented non-exhaustive).

### Two correctness fixes

`epg::Rating.value` was documented as "minimum age" but stored the raw rating
code (off by 3); the doc is fixed and `minimum_age()` added. `epg::Crid.crid_type`
had its values swapped versus TS 102 323 Table 117; corrected, and the field is
now a `CridType`.

### One new convenience: `InnerTsRecovery` (dvb-t2mi)

T2-MI → inner-TS recovery was a three-stage chain every consumer wired by hand
(`T2miPump` → `Bbheader` → `CarryOverExtractor`). `dvb_t2mi::inner_ts::InnerTsRecovery`
folds it into one `feed(&ts_packet) -> &[[u8; 188]]` driver. (This made
`dvb-bbframe` an optional dependency of `dvb-t2mi`, enabled by the existing `ts`
feature.)

## Migration

- Match the new enums (`RunningStatus`, `ServiceType`, `StreamType`,
  `CridType`, …) instead of integer literals; or call `RunningStatus::from_u8(b)`
  / `.to_u8()` to bridge to/from the wire value. The enums are `#[non_exhaustive]`
  — add a wildcard arm.
- `epg::Crid.crid_type` and the various `*_type` fields changed type; construct
  them with the enum variant.
- serde output for these fields changes from a number to the variant name
  (e.g. `"running"`), which is the intended, more-useful JSON.
- Already true in 5.0 (now documented, not new): every table serializes via the
  `Serialize` contract, and AC-3 (0x6A) / E-AC-3 (0x7A) are typed descriptors in
  `AnyDescriptor` — you do not need to hand-roll PMT/PAT encoders or walk ES
  descriptor bytes.

## Verification

Full CI gate (build, test, clippy `-D warnings`, fmt, doc `-D warnings`) passes on
stable and 1.75 MSRV, both `--all-features` and `--no-default-features`. Every new
enum has a `from_u8`→`name` test and a lossless `to_u8 ∘ from_u8` round-trip over
its value domain; the M6 carousel and broadcast-capture fixtures still round-trip
byte-identically.
