# v7.6.0 — 2026-06-20

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`). No
breaking changes. Independent crates (`dvb-pes` 0.1.1, `dvb-stream` 0.2.1,
`dvb-subtitle` 0.1.0, `mpeg-ps` 0.1.0, `scte104` 0.1.0) unchanged this tag.

## Highlights

### `dvb-si` — cpcm_delivery_signalling extension descriptor (#261)
The `descriptor_tag_extension` **0x01 cpcm_delivery_signalling_descriptor** (ETSI
TS 102 825-9 §4.1.5, Table 2) is now typed — `cpcm_version` + the version-dependent
CPCM USI `selector_bytes` — instead of falling through to `Raw`. Exposed as
`ExtensionBody::CpcmDeliverySignalling` / `ExtensionTag::CpcmDeliverySignalling`,
cross-validated by a TSDuck-compiled interop fixture.

No source changes required — additive only.
