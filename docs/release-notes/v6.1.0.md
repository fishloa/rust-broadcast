# 6.1.0 — SI output half + spec correctness pass

6.0 made the library *decode* everything; 6.1 lets it **emit** a transport
stream, adds the BBFrame/T2-MI chain conveniences, runs a spec-transcription
correctness pass (which fixed two coding bugs shipped in 6.0), and lands a
cargo-fuzz harness. A pre-release audit (md-first, adversarially verified)
confirmed the surface and drove a final round of fixes.

> **Breaking, by intent (shipped in a minor — 6.0.0 had no released consumers):**
> - The **fabricated** `ScramblingMode::DvbCsa3MinimalEnhanced` (0x04) /
>   `DvbCsa3FullyEnhanced` (0x05) variants are removed — EN 300 468 v1.19.1
>   Table 87 (the latest, definitive source) marks 0x04–0x0F *reserved*. They now
>   decode as `Reserved(_)`.
> - `DataStreamAlignmentDescriptor.alignment_type` is now the typed
>   `AlignmentType` (was raw `pub u8`); `AlignmentType` gained a `Reserved(u8)`
>   catch-all and the redundant `.alignment()` accessor was removed.

## The SI output half (#56, `dvb-si`, feature `ts`)

The crate could parse a TS into sections; now it does the inverse.

- **`mux::SectionPacketizer`** — packs serialized PSI/SI sections into aligned
  188-byte TS packets (PUSI + `pointer_field`, concatenation, per-PID continuity
  counters, 0xFF stuffing). The byte-exact inverse of `SectionReassembler`
  (ISO/IEC 13818-1 §2.4.4); round-trip-verified against the demux + the fixtures.
- **`mux::SiMux`** — a section-repetition scheduler on a caller-supplied clock.
  Per-PID `(sections, interval)` entries; `poll(now)` emits the due packets.
  Default intervals are named, spec-cited constants: NIT/BAT/SDT/EIT/TDT/TOT from
  TR 101 211 §4.4, PAT/PMT 100 ms from TS 101 154 §4.1.7, 25 ms inter-section
  floor from EN 300 468 §5.1.4.1.

## Chain conveniences (#55, `dvb-bbframe`)

- **`BbframePump`** — one infallible `feed(plp_id, df_bytes) -> &[[u8; 188]]`
  driver folding the BBFrame→inner-TS chain, per-PLP carry-over; composes the
  existing `CarryOverExtractor`, no `dvb-t2mi` dependency.
- **Decoded ISSY accessors** — `BufsUnit::multiplier_bits`,
  `SignallingKind::bufs_bits`/`bufs_bytes`/`tto_t_over_256`, per the verbatim
  EN 302 755 Annex C / EN 302 307-1 Annex D ISSY transcriptions.
- GSE handoff documentation (`TsGs::Gse` → `dvb-gse`).

## Data-driven registries (#141, `dvb-si`)

`ca_system_name` / `private_data_specifier_name` are generated at build time from
vendored TSDuck `.names` data (BSD-2-Clause, attributed) — same
`Option<&'static str>` signature, fuller and drift-free, no room to invent a value.

## Correctness pass + pre-release audit

A per-enum audit (each coding diffed against its primary spec) re-transcribed
**22 spec tables across 5 docs verbatim from the PDFs** and reconciled the codings:

- **Added** spec values that were silently folded into the catch-all:
  `ServiceType` 0x20 (HEVC UHD), `AudioType` 0x04–0x7F user-private + 0x80–0x84,
  `SubtitlingType` 0x15/0x25.
- **Removed** the fabricated `ScramblingMode` 0x04/0x05 (see breaking note).
- **Fixed** citations (`AudioType` → ISO/IEC 13818-1 §2.6.19 Table 2-63;
  `data_stream_alignment` → §2.6.10) and names (`ServiceType` 0x11 → verbatim,
  two `alignment_type` commas). The ISSY coding is now definitively documented
  for both DVB-S2 (Table D.1) and DVB-T2 (Table C.1).
- The follow-up **pre-release audit** (md-first) additionally: typed the
  `alignment_type` field, added a `TsHeader` round-trip test and a
  `CarryOverExtractor` HEM-completion-path test, and cleaned a couple of magic
  numbers / a missing module citation.

## Other

- **`ts::SectionReassembler`** no longer drops a valid near-maximal section whose
  final packet is 0xFF-stuffed (#148).
- **cargo-fuzz harness** (#49) — a workspace-excluded `fuzz/` crate with one
  target per parser entry point + a round-trip property target (1.2M execs clean
  in the smoke run). Does not touch the published crates or MSRV 1.75.

## Verification

Full CI gate (build, test, clippy `-D warnings`, fmt, doc `-D warnings`) passes on
stable and 1.75 MSRV, both `--all-features` and `--no-default-features`. The
broadcast-capture fixtures still round-trip byte-identically.
