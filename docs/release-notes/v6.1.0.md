# 6.1.0 — SI output half + spec correctness pass

6.0 made the library *decode* everything; 6.1 lets it **emit** a transport
stream, adds the BBFrame/T2-MI chain conveniences, and runs a spec-transcription
correctness pass that fixed two coding bugs shipped in 6.0.

> **Breaking, by intent (shipped in a minor):** the **fabricated**
> `ScramblingMode::DvbCsa3MinimalEnhanced` (0x04) and `DvbCsa3FullyEnhanced`
> (0x05) variants are removed — EN 300 468 Table 87 marks 0x04–0x0F *reserved*,
> so they decoded reserved wire values to invented names. They now decode as
> `Reserved(_)`. 6.0.0 had no released consumers, so this rides in 6.1.0 rather
> than forcing a major.

## The SI output half (#56, `dvb-si`, feature `ts`)

The crate could parse a TS into sections; now it can do the inverse.

- **`mux::SectionPacketizer`** — packs serialized PSI/SI sections into aligned
  188-byte TS packets (PUSI + `pointer_field`, section concatenation, per-PID
  continuity counters, 0xFF stuffing). The byte-exact inverse of
  `SectionReassembler` (ISO/IEC 13818-1 §2.4.4); round-trip-verified against the
  crate's own demux and the broadcast fixtures.
- **`mux::SiMux`** — a section-repetition scheduler on a caller-supplied clock
  (no clock dependency). Per-PID `(sections, interval)` entries; `poll(now)`
  emits packets for every entry whose interval has elapsed. Default intervals are
  named, spec-cited constants: NIT/BAT/SDT/EIT/TDT/TOT from TR 101 211
  §4.4.1/§4.4.2, PAT/PMT 100 ms from TS 101 154 §4.1.7, with the 25 ms
  inter-section floor from EN 300 468 §5.1.4.1.

## Chain conveniences (#55, `dvb-bbframe`)

- **`BbframePump`** — folds the BBFrame→inner-TS chain consumers hand-wired into
  one infallible `feed(plp_id, df_bytes) -> &[[u8; 188]]` driver with per-PLP
  carry-over state; composes the existing `CarryOverExtractor`, no `dvb-t2mi`
  dependency.
- **Decoded ISSY accessors** — `BufsUnit::multiplier_bits`,
  `SignallingKind::bufs_bits` / `bufs_bytes`, and `tto_t_over_256` (exact TTO in
  units of T/256), per the verbatim EN 302 755 Annex C / EN 302 307-1 Annex D
  ISSY transcriptions.
- GSE handoff documentation (`TsGs::Gse` → hand the data field to `dvb-gse`).

## Data-driven registries (#141, `dvb-si`)

`ca_system_name` / `private_data_specifier_name` are now generated at build time
from vendored TSDuck `.names` data (BSD-2-Clause, attributed) — same
`Option<&'static str>` signature, fuller and drift-free, with no chance for a
hand-edit/codegen to invent a registry value.

## Correctness pass (spec audit)

A per-enum audit (each coding diffed against its **primary** vendored PDF) found
that several `docs/` transcriptions were geometry-mangled and a few `.rs`
codings had drifted. **22 spec tables across 5 docs were re-transcribed verbatim
from the PDFs**, and the codings reconciled:

- **Added** decode values that were silently folded into the catch-all:
  `ServiceType` 0x20 (HEVC UHD), `AudioType` 0x04–0x7F user-private + 0x80–0x84
  (Primary/Native/Emergency/Primary commentary/Alternate commentary),
  `SubtitlingType` 0x15/0x25 (plano-stereoscopic disparity HD).
- **Removed** the fabricated `ScramblingMode` 0x04/0x05 (see breaking note above).
- **Fixed** spec citations (`AudioType` → ISO/IEC 13818-1 §2.6.19 Table 2-63;
  `data_stream_alignment` → §2.6.10) and two `alignment_type` names.
- The ISSY coding (BUFS/TTO) is now definitively documented across both DVB-S2
  (Table D.1) and DVB-T2 (Table C.1).

## Other

- **`ts::SectionReassembler`** no longer drops a valid near-maximal section whose
  final packet is 0xFF-stuffed (#148).
- **cargo-fuzz harness** (#49) — a workspace-excluded `fuzz/` crate with one
  target per parser entry point + a round-trip property target. Does not touch
  the published crates or MSRV 1.75.

## Fixes / fidelity

`#[non_exhaustive]` is preserved on the touched enums; adding the new decode
variants is non-breaking (consumers already match a wildcard). The
`ScramblingMode` removal is the only breaking item, documented above.

## Verification

Full CI gate (build, test, clippy `-D warnings`, fmt, doc `-D warnings`) passes on
stable and 1.75 MSRV, both `--all-features` and `--no-default-features`. The
broadcast-capture fixtures still round-trip byte-identically; the fuzz roundtrip
target ran 1.2M executions with no findings.
