# v7.8.0 — 2026-06-21

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`). No
breaking changes.

This tag is the **base** for a coordinated batch: it must publish first because
the companion crates below pin `dvb-common = "7.8"`. After `v7.8.0` is live on
crates.io, push the companion tags (`dvb-cc-v0.2.0`, then each
`dvb-<crate>-v0.1.0`).

## Highlights

### `dvb-si` — association_tag_descriptor (tag 0x14) (#310)

The DSM-CC `association_tag_descriptor` (ISO/IEC 13818-6 §11.4.1, transcribed
from the free ETSI TR 101 202 §4.7.7.2 Table 4.18) — the 16-bit cousin of
`stream_identifier_descriptor`, carried in the PMT `ES_info` loop. Typed
`transaction_id()` / `timeout()` accessors decode the `use == 0x0000`
(DSI-with-IOR) selector. The only in-scope descriptor gap found by an empirical
sweep of 293 TSDuck/real streams; previously fell through to
`AnyDescriptor::Unknown`. Verified against a real ETSI Hot Bird MHP capture.

### `dvb-scte35` — DVB Targeted Advertising Part 1 (#307)

The `dvb_ta` module — ETSI TS 103 752-1: the `DVB_DAS_descriptor` (tag 0xF0,
identifier `0x4456425F` "DVB_") wired into `AnySpliceDescriptor`, Compact SCTE 35
(`compact_time_signal` / `compact_splice_insert`), segmentation profiling, and
the DSM-CC stream-event payload. (Audit follow-up #318 also closed a
`descriptor_length` u8-overflow on over-long UPIDs and added TOML spec-drift
guards for `EquivalentSegmentationType` / `TimelineType`.)

### `dvb-tools` — `dump` decodes descriptor loops (#312)

`dump` human mode now walks and names the PMT (program-info + per-ES), SDT
(per-service), and NIT (network + per-TS) descriptor loops through a registry
with the PDS-scoped 0x83 logical_channel enabled for the EACEM/NorDig
`private_data_specifier`s — so LCNs show as `LOGICAL_CHANNEL` instead of
`UNKNOWN(0x83)`.

### Extensibility / code-quality audit (#313–#321)

An adversarial per-crate audit of the new modules + 6 new companion crates
(`docs/RELEASE-AUDIT.md` §3) found and fixed 6 real bugs (each with a
revert-verified regression test) and a systematic gap (7 crates lacked the #204
`label_coverage` drift-guard — all now have one). See `docs/AUDIT-LEDGER.md`.

## Companion releases (separate tags, after `v7.8.0`)

- **`dvb-cc` 0.2.0** — adds the CEA-608/708 caption **decode** layer behind a
  default-on `decode` feature (folds onto the existing `cc_data()` carriage):
  `Cea608Decoder` (pop-on/roll-up/paint-on, PACs, mid-row, CC1–CC4, XDS skip) +
  `Cea708Decoder` (CCP reassembly → service blocks → C0/C1/G0–G3 + window/pen
  model), with a typed display model. Grounded in ANSI/CTA-608-E, CTA-708-E, and
  47 CFR §79.102. `#![no_std]` + `alloc`.
- **`dvb-smpte2038` 0.1.0** — SMPTE ST 2038:2021 ANC data in TS (PES wrapper +
  ANC packet loop, 10-bit UDWs + checksum). Depends on `dvb-pes`.
- **`dvb-ule` 0.1.0** — ULE (RFC 4326 + RFC 5163): SNDU + TS-packet defragment
  with the RFC §7.2.1 payload-pointer reassembly check; typed Type/H-Type fields.
- **`dvb-simulcrypt` 0.1.0** — DVB SimulCrypt head-end signalling (ETSI TS
  103 197): ECMG↔SCS / EMMG·PDG↔MUX message + TLV parameter framing.
- **`dvb-emsg` 0.1.0** — MPEG-DASH Event Message box (`emsg`, ISO/IEC 23009-1
  §5.10.3.3) v0 + v1, with SCTE-35 scheme recognition.
- **`dvb-vbi` 0.1.0** — VBI data carriage (ETSI EN 301 775): typed Teletext / VPS
  / WSS / CC / monochrome data units.
- **`dvb-flute` 0.1.0** — ALC/LCT/FLUTE/NORM multicast object delivery (RFC
  5651 / 5775 / 6726 / 5740), incl. flag-dependent LCT header sizing and NORM
  message types (DATA/INFO/CMD/NACK/ACK).
