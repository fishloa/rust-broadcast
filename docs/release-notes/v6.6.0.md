# 6.6.0 — L1 signalling + BIOP object-carousel

A **purely additive** release — new parsers and benchmark suites. No breaking changes.

## `dvb-t2mi` — L1-pre / L1-post signalling parser (#54)

The EN 302 755 §7.2 L1 signalling layer is now fully parsed in `payload::l1`:

- `L1Pre` (Figure 25) — the L1-pre block, 32 bytes fixed, with all 13 coded
  wire fields typed via new enums: `GuardInterval`, `L1Modulation`, `T2Version`,
  `L1PostExtension`, `NumRf`, `S2Field1` and more.
- `L1PostConfigurable` (Figure 27) + `L1PostDynamic` (Figure 28) + `L1ExtBlock`
  (Table 37) + the unified `L1Post` view.
- Symmetric `Parse`/`Serialize` on every type; validated byte-exactly against
  the real `colombia-capital-t2mi.ts` broadcast capture and confirmed to
  re-serialise identically.
- Version/profile-dependent decode accessors: `PaprReduction` and `PlpCodeRate`
  dispatch on `t2_version` + `papr_*` bits.
- Lazy accessors on the existing raw payloads: `L1CurrentPayload::l1_pre()` /
  `l1_post()` (raw `l1_current_data` bytes retained, no re-allocation);
  `L1FuturePayload` dynamic-next / in-band accessors.
- 13 new `spec_tables/*.toml` TOML drift-guards (CI-enforced).
- `Error::L1Bits` added for bit-reader range errors; uses the new
  `dvb_common::bits` `BitReader`.

### `dvb-common` — `bits` module

`dvb_common::bits` provides `BitReader` / `BitWriter`, an MSB-first (DVB/MPEG
bit-order) reader/writer for dense sub-byte wire fields:

- Reads/writes 0..=64 bits per call with `BufferTooShort`-style `BitError` context.
- The writer sets *and* clears each target bit — no pre-zeroing of the output
  buffer is needed.
- Foundational infrastructure for the L1 signalling parser and any future
  tightly-packed wire format.

## `dvb-si` — BIOP object-carousel layer (#64)

`carousel::biop` implements the BIOP message layer from ISO/IEC 13818-6 §11 as
profiled by DVB TR 101 202:

- **IOR** + tagged profiles — BIOP Profile, Lite Options Profile, CORBA-CDR
  big-endian encoding; DVB alias type-ids.
- **`ObjectLocation`** / **`ConnBinder`** / **`Tap`** / **`NsapAddress`** /
  **`ServiceLocation`** typed.
- **`BiopMessage`** — `Directory`, `File`, `ServiceGateway`, `Stream`,
  `StreamEvent` variants, each with symmetric `Parse`/`Serialize`.
- **`ModuleInfo`** — with `compressed_module_descriptor` detection.
- **`ServiceGatewayInfo`** — parses the DSI `private_data` field (previously
  deferred from #50), giving callers a typed entry point into the carousel.
- **`CarouselFs`** — a virtual-file-tree walker: `resolve(path)` traverses
  the Directory/File hierarchy; `file_bytes()` returns the raw data. Composes
  with the existing `ModuleReassembler` → `BiopMessage` decode chain.
- All types covered by round-trip tests; `m6-single.ts` live-capture oracle
  validates the full chain byte-exactly.
- New `spec_tables/*.toml` TOML drift-guards for BIOP message-kind codes.

### Optional `flate2` feature

A new **default-OFF `flate2`** feature enables transparent zlib decompression
of `compressed_module_descriptor` carousel modules. Without it the compressed
module bytes are exposed raw. Default and `--no-default-features` builds are
unaffected.

## Criterion benchmark suites (#62, dev-only)

Three new benchmark suites covering hot paths across the family — all
dev-dependencies only, no API change, no MSRV impact (pinned via the workspace
Cargo.lock):

- `dvb-common/benches/crc32.rs` — `crc32_mpeg2::compute` at 188 B / 4 096 B /
  65 536 B payload sizes.
- `dvb-t2mi/benches/t2mi_hot_paths.rs` — `T2miPump::feed_ts` throughput on the
  Colombia T2-MI real-capture fixture.
- `dvb-si/benches/si_hot_paths.rs` — `SiDemux::feed` on both real fixtures,
  `AnyTableSection::parse` dispatch, `parse_loop` descriptor-loop walk, and
  `DvbText::decode`.
- `dvb-bbframe/benches/bbframe_hot_paths.rs` — `BbframePump::feed` on the TNT
  capture and `up_iter`/`NmTsIter` throughput on a synthetic NM data field.
