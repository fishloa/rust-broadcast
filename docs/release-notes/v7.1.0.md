# 7.1.0 — parser hardening, 1.81 idioms, on-demand fixtures, WASM demo

A non-breaking minor. No public API or behaviour change — internal robustness,
a Rust-1.81 idiom pass, and two repo-level additions (a fetch-on-demand test
corpus and a browser demo). MSRV stays **1.81**; build/test with `--locked`.

## Bounds-checked byte-readers (#207)

Across all five library crates (~270 read sites), manual multi-byte
`from_be_bytes([buf[i], buf[i+1], …])` index reads — and the separate length
checks guarding them — are replaced with `split_first_chunk` /
`split_last_chunk` / `first_chunk` (stable since 1.77, `core`/no_std-safe), which
fold the bounds check and the read into one atomic operation. Fixed-size
descriptor-entry loops use `chunks_exact` (each already guarded by a
`length % ENTRY_LEN` check). This removes a whole class of off-by-one / index
arithmetic from the parsers.

**Behaviour is byte-identical**, verified by the existing round-trip, serde,
fixture and spec-drift suites. DSM-CC / BIOP windowed cursor reads continue to
bound-check against the declared message/object window (not the physical
buffer), so over-read of a malformed declared length is still rejected.

The change was landed incrementally per crate and put through an adversarial
review pass; three behaviour divergences introduced during the sweep were caught
and fixed before release (an error-precedence change in the SCTE 35 section
parser, physical-vs-window bounds in the BIOP carousel readers, and a
fixed-offset CRC read in `dvb_t2mi::L1Pre::parse_with_crc` — the latter now
covered by a regression test).

## Rust 1.81 idiom cleanup (#209)

Exclusive-range match patterns (`v @ 0x03..0x40`) in the dvb-si coded-field
decoders, now that the MSRV is 1.81.

## Fetch-on-demand broadcast-capture corpus (#67)

`tools/fetch-test-streams.sh` pulls large, real third-party muxes into a
gitignored `.test-streams/` on demand (nothing vendored); the
`downloaded_streams` test runs each present capture through `SiDemux` and skips
cleanly when none are downloaded. CI / contributors can opt into the extended
corpus without bloating the repo.

## WASM browser demo (#65)

A `demo/` crate (`publish = false`, excluded from the workspace) compiles dvb-si
to WebAssembly and parses an uploaded `.ts` in the browser, plus a non-gating
GitHub Pages workflow. Not part of the published crates.

`dvb-scte35`, `dvb-conformance`, `dvb-tools`. `dvb-stream` stays on its own
cadence.
