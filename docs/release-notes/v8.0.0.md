# v8.0.0 — 2026-06-27

Lockstep major: MPEG-2 TS framing extracted to the new `mpeg-ts` crate.
Bundles the `dvb-tools` clap CLI standard (#344).

## Breaking changes

### `dvb-si`: TS framing modules removed

The MPEG-2 TS packet/adaptation/PCR/section/resync/mux/pid layer has been
extracted from `dvb-si` into the new `mpeg-ts` crate. Public modules removed:

| Removed from `dvb_si` | Replacement in `mpeg_ts` |
|---|---|
| `dvb_si::ts` | `mpeg_ts::ts` |
| `dvb_si::section` | `mpeg_ts::section` |
| `dvb_si::resync` | `mpeg_ts::resync` |
| `dvb_si::mux` | `mpeg_ts::mux` |
| `dvb_si::pid` | `mpeg_ts::pid` |

All PSI tables (PAT, PMT, CAT, TSDT, NIT, SDT, EIT, TDT/TOT, UNT, …) and
all descriptors remain in `dvb-si`. The `SiDemux` continues to work exactly
as before (it now uses `mpeg-ts` internally via the `ts` feature).

**Migration** — add `mpeg-ts` to your `Cargo.toml` and update imports:

```toml
[dependencies]
mpeg-ts = "0.1"
dvb-si  = "8"
```

```rust
// Before:
use dvb_si::ts::TsPacket;
use dvb_si::ts::SectionReassembler;

// After:
use mpeg_ts::ts::TsPacket;
use mpeg_ts::ts::SectionReassembler;
```

## Other changes

### `dvb-tools`: clap CLI standard (#344)

The `dvb-tools` CLI is rebuilt on `clap` (derive) per the workspace CLI
standard (`docs/CLI-STANDARD.md`): auto-generated `--help`/`--version`,
per-subcommand help, and proper argument validation.

### `mpeg-ts`: typed packet inspection

The `mpeg-ts` 0.1.0 release ships typed `ScramblingControl` /
`AdaptationFieldControl` enums with `name()` + `Display` (#204) and the
`scrambling_control()` / `adaptation_field_control()` accessors on both
`TsHeader` and `OwnedTsPacket`. The free helpers `iter_packets(&[u8])` and
`extract_ts_payload(&[u8])` simplify bulk-packet walking and raw payload
extraction. See `examples/iter_packets.rs`.
