# 5.0.0 — type everything, harden everything

The largest `rust-dvb` release to date and a deliberate API break across all
four crates. It finishes the project's defining promise — *typed, decoded,
spec-cited data, never raw byte slices* — hardens every parser against
attacker-controlled length fields, and opens the dispatch machinery so
third-party crates can extend it without a fork. It bundles ~50 changes and
**four adversarial 8-dimension audit rounds** since 4.3.0.

## Why a major

4.x still exposed a handful of structured loops as raw `&[u8]` ("decode it
yourself"), kept a few redundant wire-count fields that could desync from the
data they described, carried two dead legacy traits, and had section parsers
that could underflow on a malformed `section_length`. None of that is
acceptable in a library whose whole point is spec-faithful, safe decoding.
5.0 is the window to fix all of it at once.

## Headline changes

### 1. No more raw structured data

Every remaining "raw loop" is now fully typed: `target_region` /
`target_region_name`, `network_change_notify`, `T2_delivery_system`,
`audio_preselection`, `S2X_satellite_delivery_system`, `TTML_subtitling`, six
SI tables, the `enhanced_AC-3` and `linkage` conditional blocks, the DSM-CC /
UNT `compatibility_descriptor()`, the T2-MI `individual_addressing` payload, and
six extension descriptors (T2-MI, video depth range, VVC subpictures, image
icon, service prominence, SH delivery system) that previously fell through to
`ExtensionBody::Raw`. Where a field was redundant with the typed data
(`RctSection::link_count`, the SAT `ephemeris_data_count` / `metadata_flag`,
`SogiEntry`'s two presence flags), it was **removed** and is now derived on
serialize — an unrepresentable-on-wire state is no longer expressible.

### 2. Hardened against hostile input

The threat model is explicit: every length field in a transport stream is
attacker-controlled. A `section_length` smaller than header + CRC used to
underflow into an out-of-bounds slice (remote panic / DoS). All long-form table
parsers now floor it through one shared `crate::tables::check_section_length`
helper; the `extended_event` short-declared-length panic is fixed; and the SAT
`section_length` overflow guard now compares in `usize` before the `u16` cast.
Reserved-bit emission is corrected and pinned by a byte-identity conformance
test against real broadcast captures.

### 3. Third-party extensibility

A new crate (e.g. a future `dvb-scte35`) plugs its own wire types into the
existing dispatch with **zero changes to these crates** — via four runtime
registries (`DescriptorRegistry` incl. PDS scoping, `TableRegistry`,
`ExtensionRegistry`, and `dvb-t2mi`'s `PayloadRegistry`) and the now-un-sealed
`TableDef` / `DescriptorDef` / `ExtensionBodyDef` / `PayloadDef` traits.
Registered values are recovered with the inherent `downcast_ref` on the
`dyn *Object`, correct under both the default and `--no-default-features`
builds. See [`docs/extending.md`](../extending.md).

### 4. API stability for the long haul

Growth-prone enums and the `collect` outputs are `#[non_exhaustive]`; serde
camelCase is confined to the top-level dispatch / registry enums (bodies and
structs use PascalCase / snake_case); the dead legacy `Descriptor` / `Table`
traits and `Error::InvalidBcd` are gone; and the cross-crate error model is
unified into consistent structured `thiserror` variants.

## Migration

- **Error handling:** match the unified structured `Error` variants
  (`BufferTooShort`, `SectionLengthOverflow`, `InvalidDescriptor`, …) instead of
  the old per-crate shapes.
- **Removed traits:** replace `Descriptor::TAG` / `Table::TABLE_ID` / `Table::PID`
  / `.descriptor_length()` with `DescriptorDef::TAG`, `TableDef::TABLE_ID_RANGES`,
  the module-level `TABLE_ID` / `PID` consts, or `serialized_len() - 2`.
- **Renamed extension bodies:** `T2miDescriptor → T2mi`,
  `VideoDepthRangeDescriptor → VideoDepthRange`,
  `VvcSubpicturesDescriptor → VvcSubpictures`.
- **Removed fields:** stop constructing `RctSection::link_count`, SAT
  `ephemeris_data_count` / `metadata_flag`, and `SogiEntry`'s presence flags —
  they're derived from the typed data now.
- **`#[non_exhaustive]`:** add a wildcard arm when matching the dispatch enums,
  stats structs, and growth-prone field enums exhaustively.
- **serde consumers:** `epg` structs, `FunctionBody`, `IconLocation`, and
  `VbiService` now emit snake_case fields / PascalCase variant keys.
- **`dvb_common::crc32_mpeg2::TABLE`** is no longer public (use `compute()`).

## Verification

The full CI gate (build, test, clippy `-D warnings`, fmt, doc `-D warnings`)
passes on stable and the 1.75 MSRV, both `--all-features` and
`--no-default-features`. Fixture tests validate byte-identity against real
French TNT (M6 / HbbTV) and satellite captures. The extensibility surface was
validated end-to-end by an external scratch crate implementing all four `*Def`
traits on both feature arms.
