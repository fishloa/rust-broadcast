# v7.9.0 — 2026-06-22

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`). No
breaking changes. No companion-crate releases this tag.

## Highlights

### `dvb-si` — typed CPCM USI decode (#324)

The `cpcm_delivery_signalling_descriptor` (extension tag 0x01, TS 102 825-9
§4.1.5) previously exposed its CPCM Usage State Information as raw
`selector_bytes`. With ETSI TS 102 825-4 now transcribed
(`dvb-si/docs/descriptors/ts_102_825/cpcm-usi.md`), the USI is fully typed:

- **`CpcmDeliverySignalling::usi()`** → `Option<Result<CpcmUsi>>`, decoding the
  selector for `cpcm_version == 1` per TS 102 825-4 §5.4 **Table 8** — the 3-byte
  flag prefix plus every conditional field (view-window dates, view period,
  simultaneous-view count, remote-access date, CPS vector).
- New public types: **`CpcmUsi`**, **`CopyControl`** (Table 9),
  **`MoveCopyPropagation`** (Table 10), **`ViewPropagation`** (Table 11),
  **`CpsVectorEntry`** — the three coded enums carry the standard
  `name()` + `Display` label convention and `#[non_exhaustive]`.
- Symmetric `Parse`/`Serialize` with byte-exact round-trip; the CPS-vector nesting
  was confirmed against the rendered TS 102 825-4 syntax (pp. 25–26).

**Additive / non-breaking:** the published `CpcmDeliverySignalling { cpcm_version,
selector_bytes }` and its round-trip are unchanged — a non-version-1 or
non-conformant selector simply stays raw (`usi()` returns `None`/`Err`).

The other six core crates carry no functional changes (lockstep bump).
