# v8.1.0 — 2026-06-29

Lockstep minor: `dvb-common` renamed to `broadcast-common` (repository-wide
rename rust-dvb → rust-broadcast). Non-breaking — `dvb-common` continues as a
deprecated re-export shim; existing `dvb-common = "8"` deps resolve correctly.

## Overview

The shared base crate was renamed from `dvb-common` to `broadcast-common` to
align with the broader repository rename (rust-dvb → rust-broadcast). This is
purely organizational: no API changes, no behavior changes. Every crate that
depends on `dvb-common` now depends on `broadcast-common` internally.

Old code using `dvb-common` continues to work (the `dvb-common` 8.1.0 shim
re-exports `broadcast-common` in full), but **new code should migrate to
`broadcast-common`** directly.

## Migration

### Cargo.toml

```toml
# Before:
dvb-common = "8"

# After:
broadcast-common = "8"
```

### Rust imports

```rust
// Before:
use dvb_common::{Parse, Serialize, crc32_mpeg2};

// After:
use broadcast_common::{Parse, Serialize, crc32_mpeg2};
```

(If you are only using dependent crates like `dvb-si`, no import changes needed.)
