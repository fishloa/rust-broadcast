# 7.0.0 — no_std support, finished BIOP layer, full enum hardening

A **breaking** major. Three themes: embedded-ready `no_std`, completion of the
typed BIOP object-carousel layer, and breaking future-proofing (every coded
field typed, every public enum `#[non_exhaustive]`) — all the breaking work
batched into one major so the 6.x→7.0 jump is the only break.

## MSRV → 1.81

The workspace MSRV is now **1.81** (for `core::error::Error`, which makes the
`thiserror`-derived error types work under `no_std`). Build/test with `--locked`.

## `no_std` + `alloc` (#63)

The six **library** crates — `dvb-common`, `dvb-si`, `dvb-t2mi`, `dvb-bbframe`,
`dvb-scte35`, `dvb-conformance` — are now `#![no_std]` when their new **default
`std` feature** is turned off; they require only `alloc`. Parse a transport
stream on a microcontroller.

- A default-on `std` feature; `--no-default-features` = the `no_std` build.
- `HashMap`→`alloc::collections::BTreeMap` (deterministic order), `Arc`→
  `alloc::sync::Arc`, `std::*`→`core::*`/`alloc::*`.
- `f64::floor` (a std-only intrinsic) in the MJD↔calendar codec → `libm::floor`
  (bit-identical; MJD round-trips confirm).
- chrono configured `no_std + alloc` (wall-clock + std glue gate behind `std`);
  `flate2` (and its `std::io`) gate behind `std`.
- `dvb-tools` (CLI) and `dvb-stream` (tokio) remain std-only applications.
- A CI job builds every library crate for `thumbv7em-none-eabi` — the real
  `#![no_std]` proof.

## Finished BIOP object-carousel layer (dvb-si)

The BIOP message layer (ISO/IEC 13818-6 via TR 101 202) is now fully typed —
no more raw `&[u8]` for structured data:

- **`StreamMessage` / `StreamEventMessage`** (Tables 4.11 / 4.13) replace the
  raw `BiopMessage::Stream` / `StreamEvent` payloads: `DsmStreamInfo`
  (description, signed NPT duration, audio/video/data), stream taps, and the
  event-name / event-id lists. Unknown `objectKind` now errors.
- **`ServiceContext`** — `serviceContextList` is now `Vec<ServiceContext>`
  (`context_id` + data), not a raw blob, across every BIOP message.
- **`carousel_identifier_descriptor`** (tag 0x13, Tables 4.17/4.17a) — new
  descriptor: `carousel_id` + `FormatId` + the aggregated ServiceGateway-location
  `FormatSpecifier` (FormatId 0x01).

All with byte-exact round-trip + byte-anchor tests.

## Breaking future-proofing

- **Coded integer fields are now typed enums** (#210) — `binding_type`,
  `application_type`, `update_type`, `icon_transport_mode`, `identifier_type`,
  `change_type`, `range_type`, `time_plan_mode`. Each value table was verified
  against the vendored spec (no fabrication); each enum carries `name()` +
  `Display` and a `Reserved` catch-all. (`telephone connection_type` and
  `dts surround_mode` stay raw — their value tables aren't in a vendored spec.)
- **Every public enum is now `#[non_exhaustive]`** (#208), enforced by a
  per-crate `non_exhaustive_coverage` drift-guard, so future spec-value
  additions are non-breaking from here on.

## Migration

- Bump the MSRV to 1.81.
- `match` on any dvb-* public enum now needs a `_ =>` arm (`#[non_exhaustive]`).
- Read code off the new enums via `from_u8`/`name()`/`Display` instead of raw
  integers; `BiopMessage::Stream`/`StreamEvent` and `serviceContext` are typed.
- For `no_std`: depend with `default-features = false` and re-enable features as
  needed; `std` is a feature now.
