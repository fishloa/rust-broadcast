Minor. Adds the feasible TR 101 290 Priority-3 conformance indicators to `dvb-conformance`. Additive — no breaking API change. The other five lockstep crates (`broadcast-common`, `dvb-si`, `dvb-t2mi`, `dvb-bbframe`, `dvb-tools`) move to 8.6.0 in lockstep with no functional change.

## dvb-conformance — TR 101 290 Priority-3 indicators (#732)

The monitor previously implemented only 3.2 (SI_repetition, max-interval) of Priority 3, and lumped 3.1 / 3.3–3.10 as "out of scope". Six feasible Priority-3 indicators are now implemented (reusing the existing lazily-armed SI-timer + referenced-PID machinery — no new architecture), taking the indicator count from 13 to 19:

- **3.1 `NitError`** — wrong table_id (∉ {0x40, 0x41, 0x72}) on PID 0x0010; NIT_actual (0x40) absent > 10 s.
- **3.4 `UnreferencedPid`** — a PID outside {PAT, CAT, well-known SI PIDs, reserved 0x0002–0x000F, null} not referenced by a PMT within the persistence threshold.
- **3.5 `SdtError`** — wrong table_id (∉ {0x42, 0x46, 0x4A, 0x72}) on PID 0x0011; SDT_actual (0x42) absent > 2 s.
- **3.6 `EitError`** — wrong table_id (∉ {0x4E, 0x4F, 0x50–0x5F, 0x60–0x6F, 0x72}) on PID 0x0012; EIT P/F actual (0x4E) absent > 2 s.
- **3.7 `RstError`** — wrong table_id (∉ {0x71, 0x72}) on PID 0x0013.
- **3.8 `TdtError`** — wrong table_id (∉ {0x70, 0x72, 0x73}) on PID 0x0014; TDT (0x70) absent > 30 s.

All grounded in the new faithful spec transcription `dvb-conformance/docs/tr_101_290.md` (TR 101 290 v1.4.1 Table 5.0c, rendered from the PDF); each indicator cites its clause. `Indicator` is `#[non_exhaustive]` so the additions are non-breaking. 20 new biting tests (bad-table_id trip + absence-timeout trip + clean-stream negative control per indicator).

### Still excluded — now split by reason (README)
- **Not computable** (need the ISO/IEC 13818-1 T-STD buffer model / hardware arrival timing; this monitor is sans-IO, caller-supplied clock): 2.4 PCR_accuracy, 3.3 Buffer_error, 3.9 Empty_buffer_error, 3.10 Data_delay_error.
- **Feasible, deferred**: the 25 ms minimum-gap dimensions, the `_other` repetition sub-clauses (3.1.b/3.5.b/3.6.b), and 3.6.c EIT P/F pairing.

## Compatibility

MSRV 1.86. Additive `#[non_exhaustive]` enum variants + a new spec-transcription doc — recompile, no source changes required.
