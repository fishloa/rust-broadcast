# v7.5.0 — 2026-06-19

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`). No
breaking changes. The independently-versioned crates `dvb-pes` (**0.1.1**) and
`dvb-stream` (**0.2.1**) ship alongside for their new examples; `dvb-subtitle`
(0.1.0) and `mpeg-ps` (0.1.0) released separately on their own tags.

## Highlights

### `dvb-si` — protection_message extension descriptor (#258)
The `descriptor_tag_extension` **0x18 protection_message_descriptor** (ETSI
TS 102 809 §9.3.3, Table 40) is now fully typed — the protected `component_tag`
list — instead of falling through to `Raw`. Exposed as
`ExtensionBody::ProtectionMessage` / `ExtensionTag::ProtectionMessage`, with the
4-bit `component_count` guarded against overflow and reserved bits preserved for
byte-exact round-trip. Cross-validated by a TSDuck-compiled interop fixture.

### Runnable examples on every library crate (#253, #255)
Each library crate now ships **two runnable examples** (a basic quickstart + an
advanced real-capture walkthrough), and the example source is **embedded into the
crate-root docs so it renders on docs.rs**:

- `dvb-common` — CRC-32/MPEG-2 + BCD; implementing the `Parse`/`Serialize` contract.
- `dvb-si` — build/parse a PAT; demux a capture and list services.
- `dvb-t2mi` — parse a T2-MI header; pump T2-MI from a capture.
- `dvb-bbframe` — parse a BBHEADER; walk a real DVB-S2 capture.
- `dvb-scte35` — parse a `splice_insert`; walk descriptors + round-trip.
- `dvb-conformance` — run the TR 101 290 monitor; priority/indicator breakdown.
- `dvb-pes` (0.1.1) — parse one PES packet; extract a PTS timeline from a capture.
- `dvb-stream` (0.2.1) — drive `SectionStream`; tally tables + resync stats.

CI now builds `--examples` workspace-wide.
