# v8.5.0 — 2026-07-21

Lockstep release of the six DVB core crates (`broadcast-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-conformance`, `dvb-tools`). Additive minor,
riding the `mpeg-ts` 0.3.0 compatibility cycle (issue #663, the
multimux-hub epic).

## Highlights

- **`mpeg-ts` 0.3 compatibility.** `dvb-si`, `dvb-conformance`, and
  `dvb-tools` widen their internal `mpeg-ts` dependency from `0.2` to `0.3`
  to pick up `mpeg-ts` 0.3.0's `SectionPacketizer` → `SectionPacketiser`
  (and `packetize`/`packetize_into` → `packetise`/`packetise_into`) British-
  spelling rename. `mpeg-ts` is a **private** dependency of all three (used
  only internally, behind `dvb-si`'s `ts` feature and inside
  `dvb-conformance`'s test helpers) — this is **not** a breaking change for
  any consumer of these crates; no public API changed.
- `dvb-conformance`'s own test helpers were updated to the renamed
  `mpeg_ts::mux::SectionPacketiser`/`packetise` identifiers.
- `broadcast-common`/`dvb-t2mi`/`dvb-bbframe` carry no functional changes
  this release; they bump in lockstep with `dvb-si`.

## Compatibility

Additive — no breaking changes to any of the six crates' own public API.
MSRV unchanged (1.86). All inter-crate dependency references
(`version = "8"`) continue to resolve.
