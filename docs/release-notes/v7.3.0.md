# 7.3.0 — deferred DVB-MHP descriptors

A non-breaking minor. Headline: the three DVB-MHP descriptors `dvb-si` had
deferred for lack of a transcribed syntax table are now typed (#227). Lockstep
across the 7 core crates; MSRV stays **1.81**; build/test with `--locked`.

## Deferred DVB-MHP descriptors (#227)

Transcribed from **ETSI TS 102 727 V1.1.1** (DVB-MHP 1.1.3) into a new
spec-cited markdown source (`dvb-si/docs/ts_102_727_mhp.md`, BlazeDocs OCR with a
PDF spot-check), then implemented with the usual symmetric `Parse`/`Serialize` +
round-trip + byte-identical tests:

- **AIT application-descriptor namespace** (decoded by `AnyAitDescriptor`):
  - `dvb_j_application` (tag `0x03`, Table 84 §10.9.1) — a run of length-prefixed
    parameter strings (`parameters: Vec<&[u8]>`, kept raw since Java parameter
    strings are opaque, not DVB Annex A text). Now
    `AnyAitDescriptor::DvbJApplication` (previously `Unknown`).
  - `dvb_j_application_location` (tag `0x04`, Table 85 §10.9.2) — `base_directory`
    / `classpath_extension` / `initial_class`, modelled as `DvbText` to match the
    sibling `simple_application_location` (0x15). Now
    `AnyAitDescriptor::DvbJApplicationLocation` (previously `Unknown`).
- **Extension descriptor** (decoded by `ExtensionBody`):
  - `xait_pid` (`descriptor_tag_extension` `0x0C`, Table 95 §10.17.3) — the
    16-bit `xait_PID` (`uimsbf`), stored and round-tripped verbatim (Table 95
    defines no reserved-bit split). Now `ExtensionBody::XaitPid` /
    `ExtensionTag::XaitPid` (previously `Raw`), with an `is_valid_pid()` helper
    for the `<= 0x1FFF` PID semantic.

Each is wired into its `declare_*!` dispatch list (so the tag↔`*Def::TAG` drift
test covers it) and verified end-to-end against a real **Hot Bird MHP** mux: the
`hotbird-mhp` capture carries `dvb_j` `0x03`/`0x04` on AIT PIDs `0x1EC6`/`0x1EC7`,
which now decode as the typed variants (2 of each, 0 left `Unknown`).

## Upgrading

Purely additive — new typed variants on the (already `#[non_exhaustive]`)
`AnyAitDescriptor` and `ExtensionBody` enums, and a new `DvbText`-typed body on a
not-previously-implemented descriptor. Code that matched `Unknown`/`Raw` for
these tags will now see the typed variants. No other crate changed (lockstep
version bump only).
