# v7.4.0 — 2026-06-18

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`).
`dvb-stream` stays on its own cadence (0.2.0). No breaking changes.

## Highlights — `dvb-si` descriptor coverage epic (#240)

Completes the **MPEG-2 systems descriptor set (ISO/IEC 13818-1 / ITU-T H.222.0
§2.6, tags 0x02–0x38)** plus DVB extension mop-up and NorDig private descriptors.
Previously these fell through `AnyDescriptor` to the unknown arm.

- **MPEG-2 simple descriptors 0x02–0x12** (#236): video_stream, audio_stream,
  hierarchy, target_background_grid, video_window, system_clock,
  multiplex_buffer_utilization, copyright, maximum_bitrate, smoothing_buffer,
  STD, IBP.
- **MPEG-2 codec descriptors** (#237): AVC_video (0x28), HEVC_video (0x38),
  MPEG-2_AAC (0x2B), AVC_timing_and_HRD (0x2A), MPEG-4 video/audio (0x1B/0x1C).
- **MPEG-2 systems — fixed/opaque** (#237): IOD, SL, FMC, External_ES_ID,
  Muxcode, FmxBufferSize, MultiplexBuffer, metadata_STD, FlexMuxTiming,
  auxiliary_video_stream, SVC_extension, MVC_extension.
- **MPEG-2 systems — conditional + J2K** (#237): content_labeling,
  metadata_pointer, metadata, J2K_video — nested optional blocks modelled as
  typed `Option` sub-structs.
- **DVB extension descriptors** (#238): CP (0x02), CP_identifier (0x03),
  CI_ancillary_data (0x14).
- **NorDig private descriptors** (#239): NorDig logical_channel v1 (tag 0x83,
  14-bit LCN) and v2 (tag 0x87, channel-list grouped), via
  `DescriptorRegistry::with_nordig_lcn()`. Fixes a latent misparse where the
  EACEM/D-Book **10-bit** LCN layout was applied to NorDig's **14-bit** form on
  the shared tag 0x83.

New typed enums (with the `name()` + `Display` convention): `FrameRateCode`,
`HierarchyType`, `AacAdditionalInfo`, `HdrWcgIdc`, `ContentTimeBaseIndicator`,
`MetadataFormat`, `MpegCarriageFlags`, `DecoderConfigFlags`.

### Cross-tool validation
Two **TSDuck interop fixtures** (PMTs compiled by `tstabcomp`, the reference DVB
toolkit) decode byte-exact against our parsers — catching, among other things, a
HEVC `HDR_WCG_idc` bit-position bug that round-trip tests structurally could not.

## Spec docs reorganised (#234, #235)
All spec transcriptions now live as **one table per file** in a clean
`<crate>/docs/{tables,descriptors,enums,text}/<spec>/` tree, with each drift-guard
`.toml` co-located beside its `.md`, across all 4 doc-bearing crates.

## Documentation tooling
Per-crate docs.rs metadata (`all-features` + `--cfg docsrs`) and `doc_cfg`
enablement so feature-gated API renders on docs.rs; added `SECURITY.md` and the
`docs/RELEASE-DOCS.md` release-documentation standard.

## Other crates
Lockstep version bump only; no functional changes.
