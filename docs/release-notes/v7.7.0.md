# v7.7.0 — 2026-06-20

Additive lockstep minor across the 7 core crates (`dvb-common`, `dvb-si`,
`dvb-t2mi`, `dvb-bbframe`, `dvb-scte35`, `dvb-conformance`, `dvb-tools`). No
breaking changes.

Shipped alongside this tag (independent crates): **`dvb-ci` 0.1.0** (new) and
**`mpeg-ps` 0.1.1** (patch) — see their own CHANGELOGs.

## Highlights

### `dvb-si` — DTS audio extension descriptors (#263)

Three `descriptor_tag_extension` values that previously fell through to
`ExtensionBody::Raw` are now fully typed, sourced from the already-vendored
ETSI EN 300 468 V1.19.1 Annexes G and L:

- **0x0E DTS-HD** — Annex G.3, Tables G.6–G.10. Typed `substream_info()` blocks
  with `SamplingFrequency`, and `AssetInfo` (optional `component_type` and
  `ISO_639_language_code`).
- **0x0F DTS-Neural** — Annex L.1, Table L.1. `config_id` + `additional_info`.
- **0x21 DTS-UHD** — Annex G.5, Tables G.15–G.17. `DecoderProfileCode`,
  `FrameDurationCode`, `MaxPayloadCode`, `StreamIndex` + `codec_selector`, with
  new public enums `FrameDurationCode` (Table G.16) and `MaxPayloadCode`
  (Table G.17).

Exposed as `ExtensionBody::DtsHd` / `DtsNeural` / `DtsUhd` (and the matching
`ExtensionTag` variants). This brings the typed extension-descriptor registry to
**30 tag_extensions**.

## Companion releases

- **`dvb-ci` 0.1.0** — new crate: DVB Common Interface (ETSI EN 50221) wire
  protocol — APDU/resource objects (ca_info, ca_pmt, ca_pmt_reply,
  application_info, resource_manager, date_time, mmi_close), ASN.1 length codec,
  SPDU session + TPDU transport framing, and a `build_ca_pmt(&PmtSection, …)`
  builder. `#![no_std]` + `alloc`.
- **`mpeg-ps` 0.1.1** — fix: `program_stream::parse_pack` integer-underflow panic
  on adversarial input (found by cargo-fuzz); now clamps via `saturating_sub`.
