# 7.2.0 ‚Äî context-aware descriptor decoding, parser hardening

A non-breaking minor. Headline: **context-aware descriptor decoding** for the AIT
and private-LCN namespaces (#211), plus a parser panic-class safety pass (#215/
#216/#217) and real-stream test fixtures (#218). Lockstep across the 7 core
crates; MSRV stays **1.81**; build/test with `--locked`.

## Context-aware descriptor decoding (#211)

DVB reuses low descriptor-tag values with different meanings depending on context.
This release types the two most common cases:

- **AIT application-descriptor namespace** (ETSI TS 102 809 ¬ß5.3). The AIT loops
  use a separate descriptor namespace from EN 300 468 SI, so a new dispatcher
  decodes them: `AnyAitDescriptor` + `parse_ait_loop` + `AitDescriptorLoop`
  (mirroring the SI `declare_descriptors!`/`DescriptorDef` pattern, with a drift
  test). Typed modules: `application` (0x00), `application_name` (0x01),
  `transport_protocol` (0x02, with a typed OC/HTTP `selector()`),
  `external_application_authorisation` (0x05), `simple_application_location`
  (0x15), `application_usage` (0x16), `simple_application_boundary` (0x17). New
  `AitApplication::ait_descriptors()` / `AitSection::common_ait_descriptors()`
  accessors walk the loops in this namespace. (`dvb_j` 0x03/0x04 deferred ‚Äî MHP.)
- **PDS-scoped logical_channel (LCN)**. `DescriptorRegistry::with_logical_channel_for_pds(pds)`
  dispatches the private `logical_channel` (0x83) to the typed
  `AnyDescriptor::LogicalChannel` **only** when the active `private_data_specifier`
  (0x5F) matches ‚Äî 0x83 under no/other PDS stays `Unknown`. New `PDS_EACEM`
  (0x28) / `PDS_NORDIG` (0x29) constants. The `dvb-tools` `services` command now
  resolves LCNs via this path.

Both validated against a real French HbbTV mux (3 AIT sections decoded;
7 logical_channel descriptors / 59 LCN entries) ‚Äî see #218.

## Parser panic-class hardening (#215 / #216 / #217)

Malformed-input safety, surfaced by an internal per-crate audit. No behaviour
change for well-formed input.

- **#215** `dvb-t2mi` L1-Future: `l1_dynamic_next2` / `inband_loop` no longer
  panic when a wire length field exceeds the buffer (`data[offset..]` ‚Üí
  `data.get(offset..)` ‚Üí `BufferTooShort`).
- **#216** `dvb-scte35`: `SpliceInfoSection::parse` no longer underflows /
  panics on `section_length < 1`.
- **#217** workspace: open-ended `slice[pos..].split_*_chunk` cursor reads ‚Üí
  `slice.get(pos..)` (panic‚Üíerror; windowed reads untouched).

## Real-stream test fixtures (#218)

`tools/fetch-test-streams.sh` gains fetch-on-demand entries (astra-sgt LCN,
russia-t2mi-scte35, hotbird-mhp) and skip-if-absent tests; the AIT and LCN tests
are verified against the already-fetched French capture.
