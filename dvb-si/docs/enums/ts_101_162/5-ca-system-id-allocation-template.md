# Table 5: CA_System_ID allocation template

_Source: specs/etsi_ts_101_162_v01.09.01_dvb_identifiers.pdf (PDF pp.12-44). NB: this is a registration-PROCESS spec (per-identifier registration/allocation templates + domain-name registries); live value allocations are maintained at the DVB online registry, not fixed here._


|  CA_System_ID | CA system specifier  |
| --- | --- |
|  0x0000 | Reserved  |
|  0x0001 to 0x00FF | Reserved for registration to standardized systems through the DVB Project Office (see http://www.dvbservices.com)  |
|  0x0100 to 0xFFFF | Reserved for general registration through the DVB Project Office (see http://www.dvbservices.com)  |

In the standardized systems registration range, allocations shall only be made for Conditional Access systems which are defined and/or adopted as such by DVB, and which are fully described in a publicly available document from a recognized standardization body.

In the general registration range, allocations shall only be made to bona fide Conditional Access system vendors. Applicants need to demonstrate that the vendor is proposing a registration for a legitimate Conditional Access product.
