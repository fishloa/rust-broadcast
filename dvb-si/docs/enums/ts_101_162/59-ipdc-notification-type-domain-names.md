# 10.3 Root_of_Trust_ID

# 10.3.0 Root_of_Trust_ID registration principles

The 18Crypt profile of DVB-IPDC Service Purchase and Protection (SPP) uses a public-key infrastructure (PKI) to manage authorization, authentication, data integrity, and certificate revocations. The Root_of_Trust_ID globally and uniquely identifies a trust-centre organization that is responsible for issuing and managing certificates [i.19].

Root_of_Trust_ID values shall be allocated only to bona fide trust-centre organizations. Applicants need to demonstrate that the vendor is proposing a registration for a legitimate Root-of-Trust product.
